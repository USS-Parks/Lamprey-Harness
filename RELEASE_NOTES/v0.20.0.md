# Lamprey v0.20.0

This release adds Steering and a durable follow-up Queue to the running chat
composer.

## What changed

- Press Enter while a turn runs to steer that same turn at its next safe model
  boundary.
- Press Tab to place the follow-up in the next-turn Queue.
- Accepted Steering appears as a quiet pending row above the composer. Stop is
  a separate compact control.
- Edit, reorder, send, or delete queued follow-ups without leaving the chat.
- Follow-up state survives renderer reloads and application restarts.
- Exact turn IDs and idempotent client IDs make completion races visible instead
  of silently converting a failed Steer into a new turn.
- Steering during a tool call waits until the tool side effect and result are
  complete. Steering a waiting child agent keeps the existing child identity.
- Turn-control activity records outcomes and counts without copying prompt text
  or attachment data into the audit log.

## Downloads

- `Lamprey-x64.exe` — Windows x64 installer
- `Lamprey-x64.zip` — Windows x64 portable package
- `Lamprey-arm64.dmg` — macOS Apple silicon
- `Lamprey-x86_64.AppImage` — Linux x64

The Windows installer and macOS DMG are unsigned. Windows may show a SmartScreen
warning; macOS requires the standard right-click → Open first-launch path.

## Verification

The complete automated twenty-row Steering/Queue contract matrix passes. The
owner has tested the core Lamprey Steering path. The full paired replay against
the pinned Codex desktop build remains open, so v0.20.0 does not make a blanket
current-Codex parity claim.
