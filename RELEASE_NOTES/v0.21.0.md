# Lamprey v0.21.0

This release adds one coherent task and thread control surface for users and models.

## What changed

- Inspect conversations, historical forks, agent runs, identities, and turns through one
  bounded task graph without replacing their canonical stores.
- Use `list_tasks`, `read_task`, and cancellable/event-driven `wait_tasks` from chat.
- Send durable next-turn Queue messages, steer an exact active turn, or interrupt it without
  a stale Steering action silently becoming Queue.
- Fork a task at a specified completed turn, with source conversation/turn backlinks and an
  optional newly isolated worktree.
- Rename, pin, archive, close, restore, preview descendant impact, and permanently delete a
  task tree through distinct recoverable/destructive paths.
- Use the same controls in Activity, including parent/child links, live status, unread
  counts, bounded waits, Steering, interrupt, lifecycle actions, and two-step deletion.
- Persist historical fork identity in migration v22 and recoverable close state in v23.

## Downloads

- `Lamprey-x64.exe` — Windows x64 installer
- `Lamprey-x64.zip` — Windows x64 portable package
- `Lamprey-arm64.dmg` — macOS Apple silicon
- `Lamprey-x86_64.AppImage` — Linux x64

The Windows installer and macOS DMG are unsigned. Windows may show a SmartScreen warning;
macOS requires the standard right-click → Open first-launch path.

## Verification

The M2 mechanical gates cover graph construction, waiting, delivery races, historical
forking, lifecycle recovery/deletion, UI wiring, native SQLite migrations, production
build, and renderer/bundle integrity. The packaged owner GUI playbook remains open, so
v0.21.0 claims M2 implementation completion rather than blanket current-Codex parity.
