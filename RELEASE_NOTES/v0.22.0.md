# Lamprey v0.22.0 — Local release notes

This local version cut adds first-class inline visualizations and direct, revision-safe artifact
editing. It has not been pushed, tagged, or published.

## What changed

- Persist stable artifacts, immutable revisions, exact-range annotations, sandbox policy,
  export metadata, and message/conversation provenance in migrations v24–v26.
- Create Mermaid, chart, table, SVG, HTML, JSX, and React visualizations through strict lazy
  tools with bounded type-specific validation and accessible text alternatives.
- Render Mermaid/chart/table/SVG safely inline; keep interactive source out of the chat DOM and
  open it only in Lamprey's isolated artifact sandbox.
- Expand/collapse, open, edit, and export visualization cards with loading/error/ready states.
- Select exact Markdown/code/artifact source ranges, ask Lamprey for a non-destructive proposal,
  preview before/after source, and accept or reject without losing the prior revision.
- Persist assistant/user actor provenance separately for proposals, accepted revisions, and
  annotations; stale proposals fail closed as conflicts.
- Give new and legacy message documents deterministic editable artifact identities and resolve
  the latest accepted revision for open/copy/download.
- Show visualization generation, artifact edits, and file opens in Activity with
  queued/running/complete/error states and no false-success completion.

## Verification boundary

Focused prompt gates cover exact/native SQLite schema and stores, validation, tool metadata,
renderer/sandbox isolation, range/conflict/diff/accept/reject behavior, activity mapping, and
open-failure negatives. VA-6 records the full repository gate in
`PLANNING/CJ26_ARTIFACTS_AFTER.md`.

The packaged owner visualization/editing playbook remains open. v0.22.0 is a local
implementation-complete target only; downloads remain on the published v0.21.0 release until
the owner separately authorizes publication.
