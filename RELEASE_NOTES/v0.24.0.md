# Lamprey v0.24.0 — MCP resources and hosted sessions

This release adds MCP resource/template browsing and generic hosted OAuth sessions.

## What changed

- List paginated resources and URI templates from any connected server that advertises the MCP resources capability.
- Read an exact server/URI pair through strict lazy model tools with canonical content blocks and the existing large-result spill valve.
- Connect hosted MCP servers over Streamable HTTP with OAuth 2.1, PKCE, encrypted keychain storage, reauthorization, expiry, and reconnect state.
- Require explicit consent before hosted authorization, URL-mode elicitation navigation, or opening an external HTTP(S) resource.
- Browse resources/templates and hosted auth progress in Customize → Connectors, including plugin-owned connectors.
- Render text safely, allow only known raster image previews, and keep SVG or other binary content metadata-only.
- Record metadata-only session, consent, and resource-change activity without tokens, authorization URLs, or resource bodies.

## Verification boundary

MR-1 through MR-4 passed their focused type, lint, unit, build, and renderer gates. MR-5 records the full repository gate in `PLANNING/CJ26_MCP_AFTER.md`. The local fixture and real hosted-provider OAuth playbooks remain open, so v0.24.0 claims local implementation completion without claiming live hosted-provider parity.
