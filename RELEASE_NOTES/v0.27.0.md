# Lamprey v0.27.0 — Provider Discovery Expansion v2

Released 2026-07-19.

## Added

- API-key acceptance for AIHubMix, FreeLLMAPI, Cohere, MiniMax, NVIDIA API Catalog/NIM,
  GitHub Models, SambaNova, SiliconFlow, Reka, SEA-LION, DeepInfra, Hyperbolic, Perplexity
  Sonar, Sarvam AI, and Inception Labs.
- Current first-party Kimi, Grok 4.20, direct Gemma 4, Cohere, MiniMax, Reka, SEA-LION,
  Perplexity, Sarvam, and Mercury model entries.
- Provider-owned live-catalog strategies for standard OpenAI lists, bare arrays, GitHub's
  catalog, DeepInfra's typed catalog, and providers without a compatible chat-model list.
- Filtered bulk import for large live catalogs. Colliding local IDs are namespaced without
  changing the model ID sent on the wire.
- Editable FreeLLMAPI base URL for local or remote self-hosted instances.
- MiniMax `reasoning_details` preservation in Lamprey's reasoning-audit channel.

## CI repair

- Windows PowerShell AST integration assertions now run only on Windows. Non-Windows CI
  explicitly verifies the production policy's fail-closed `uninspectable` result when that
  parser is unavailable.

## Safety and honesty

- Imported models default to tools, vision, and reasoning disabled until capability evidence
  is supplied.
- Media, embedding, reranking, deprecated, and non-chat catalog entries are excluded where
  the provider exposes type metadata.
- Providers without stored user keys remain pending-key; this release does not claim live
  authentication success for credentials it did not possess.
