# Lamprey v0.27.1 — Direct-provider routing hotfix

Released 2026-07-19.

## Direct provider authority

- All 70 pinned models route through their named provider and independently stored API key.
- No pinned model uses OpenRouter as a broker. OpenRouter remains available only when a user
  explicitly saves an OpenRouter key and imports that live catalog.
- Kimi K3 routes directly to Moonshot, Grok to xAI, Gemma to Google AI, Claude to Anthropic,
  GPT to OpenAI, and every specialist or host model to that provider's own key namespace.
- Retired broker aliases for Gemma, Claude, GPT, Grok, and Kimi migrate to direct equivalents.
  Unsupported stale defaults fail to a known catalog model instead of an accidental route.

## Model menu repair

- Saving or deleting a provider key in Settings immediately synchronizes the chat model menu.
- Opening the menu refreshes provider-key state, so a stored Moonshot key unlocks Kimi K3
  without restarting Lamprey.
- Large catalogs are bounded to the viewport, scroll normally, and reveal the active model.
- Menu rows expose radio-menu semantics and no longer inherit form submission behavior.

## Verification

- A catalog invariant requires zero pinned `openrouter` models.
- Direct Gemma and current Moonshot rosters remain exact-source locked.
- Viewport, active-row, key-refresh, and Settings synchronization wiring are regression tested.
